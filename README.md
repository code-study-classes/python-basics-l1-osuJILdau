## 1 задача: Расчет итогового рейтинга студента

Вам необходимо разработать часть логики для балльно-рейтинговой системы. Система получает массив баллов студента за семестр. Нужно отбросить самую низкую и самую высокую оценку (чтобы исключить случайности), а для оставшихся баллов вычислить среднее значение.

### Условия

- Создать список из 7–10 случайных оценок (от 1 до 100).
- Найти минимальный и максимальный балл в списке.
- Удалить найденные экстремумы из списка.
- Вычислить среднее арифметическое оставшихся баллов и вывести его на экран.

### Пример вывода программы

```bash
Исходные баллы: [45, 82, 12, 90, 60, 33, 75]
Удаляем минимум (12) и максимум (90).
Оставшиеся баллы: [45, 82, 60, 33, 75]
Средний рейтинг: 59.0
```

## 2 задача: Генерация расписания с модульным контролем

Вам необходимо написать скрипт для конструктора учебных планов. Семестр длится 16 недель. Каждую неделю у студента 4 часа занятий, но каждая 4-я неделя - это модульный контроль (0 часов обычных занятий, но фиксируется слово `"Контроль"`).

### Условия

- Создать пустой список `curriculum`.
- Использовать цикл для прохождения по 16 неделям.
- Если номер недели кратен 4, в список добавляется строка `"Контроль"`.
- В остальные недели в список добавляется число `4`.
- Вывести итоговый учебный план.

### Пример вывода программы

```bash
Учебный план на семестр:
[4, 4, 4, 'Контроль', 4, 4, 4, 'Контроль', 4, 4, 4, 'Контроль', 4, 4, 4, 'Контроль']
```

## 3 задача: Массовое обновление цен со скидкой

Вам необходимо реализовать алгоритм для бэкенда интернет-магазина. Грядет распродажа, и нужно обновить массив цен. Скидка 20% применяется только к тем товарам, которые стоят строго больше 5000 рублей. Цены остальных товаров остаются без изменений.

### Условия

- Создать список `prices` из нескольких цен (например, 1500, 6000, 4999, 12000, 800).
- Использовать генератор списков (list comprehension) для создания нового массива `new_prices`.
- Внутри генератора использовать тернарный оператор для проверки условия (применять ли умножение на 0.8).

### Пример вывода программы

```bash
Старые цены: [1500, 6000, 4999, 12000, 800]
Новые цены: [1500, 4800.0, 4999, 9600.0, 800]
```

## 4 задача: Поиск пересечений в базах данных (Изоляция дубликатов)

Вам необходимо объединить данные из двух систем. У нас есть два списка с идентификаторами пользователей (ID). Нам нужно найти только тех пользователей, которые присутствуют в **обоих** списках, и сохранить их в новый массив без дубликатов.

### Условия

- Даны два списка ID: `db1 = [101, 102, 105, 108, 110]` и `db2 = [105, 199, 101, 115, 105]`.
- Создать пустой список `common_users`.
- Пройти циклом по первому списку. Если элемент присутствует во втором списке И его еще нет в итоговом списке `common_users`, добавить его.
- Использовать только циклы и оператор проверки вхождения. Запрещено использовать множества (`set`).

### Пример вывода программы

```bash
Совпадающие ID: [101, 105]
```

## 5 задача: Сглаживание графика загрузки сервера (Скользящее среднее)

Вам необходимо написать скрипт для панели мониторинга (дашборда). Дан массив показателей нагрузки на сервер за 10 часов. График слишком "дерганый". Чтобы увидеть реальный тренд, нужно сгладить данные методом скользящего среднего с окном в 3 часа (каждый новый элемент - это среднее значение текущего часа и двух последующих).

### Условия

- Дан список `load = [10, 50, 20, 80, 30, 90, 40, 60, 10, 20]`.
- Создать пустой список `smoothed_load`.
- Пройти циклом по массиву. На каждом шаге брать срез из 3 элементов: текущий `i`, следующий `i+1` и `i+2`.
- Вычислить их среднее арифметическое (можно округлить через `int()`) и добавить в итоговый список.
- **Критический нюанс:** цикл должен остановиться за 2 шага до конца исходного массива, иначе вы выйдете за его границы (`IndexError`).

### Пример вывода программы

```bash
Исходная нагрузка: [10, 50, 20, 80, 30, 90, 40, 60, 10, 20]
Сглаженный тренд: [26, 50, 43, 66, 53, 63, 36, 30]
```
